Project Overview

A two-part analysis combining rigorous EDA with a production-style ML pipeline.

🔍

Exploratory Analysis

Data cleaning, quality documentation, feature engineering, and interactive visualizations (plotly) across billing, demographics, conditions, and utilization patterns. Statistical tests: chi-square, Pearson correlation, multivariable regression.

🤖

Machine Learning Pipeline

Binary classification (Abnormal test result) using tidymodels. Logistic Regression vs. Random Forest with SMOTE resampling, hyperparameter tuning, ROC-AUC evaluation, and feature importance via vip.

🔐

Privacy-First Practice

Even on synthetic data, direct patient identifiers (name, doctor, hospital, room) are excluded from all preview tables and model inputs — modeling professional healthcare data handling standards.

📋

Reproducible Workflow

Fully documented R Markdown notebooks, version-controlled data, and a step-by-step reproduction guide. Anyone can re-run the analysis from source in minutes.

Key Findings

Selected insights from the exploratory analysis.

💵 Billing

Median billing is ~$25,000 per encounter, right-skewed with a meaningful tail of high-cost cases. Emergency admissions drive higher median billing across nearly every condition.

🏥 Conditions

Arthritis, Cancer, and Obesity have the highest median billing. Diabetes and Hypertension show the highest rates of Abnormal test results — a signal for care pathway prioritization.

📊 Statistical Tests

Chi-square confirms admission type mix differs significantly by condition (p < 0.001). Age has a weak positive correlation with billing (r ≈ 0.01) — condition and admission type matter more.

📅 Utilization

Weekday admissions are nearly uniformly distributed, suggesting this dataset captures a general inpatient population rather than a primarily elective-surgery facility.

🧠 Interpretation

The near-chance AUC is an honest and important finding. Administrative features (demographics, condition, insurance) have weak predictive signal for test outcomes in this dataset. This demonstrates the ability to correctly diagnose and communicate a null result — a critical skill in applied data science. Real-world improvement would require clinical variables (vitals, labs, procedure codes).

Reproduce This Analysis

Clone the repo and re-run in RStudio in minutes.

# 1. Clone the repository
git clone https://github.com/DuyThucCao/healthcare-data-analysis.git

# 2. Install R dependencies

  "tidyverse", "tidymodels", "janitor", "lubridate",
  "plotly", "DT", "ranger", "themis",
  "vip", "knitr", "rmarkdown", "kableExtra"
